# Mobile App - Razorpay Payment Integration

## Overview
Implemented Razorpay payment integration for paid courses in the mobile app, matching the functionality available in the web version.

## Changes Made

### 1. Updated Course Details Page
**File:** `jobzeeMobile/app/course-details.jsx`

- Modified `handleEnroll` function to redirect to payment page for paid courses
- Free courses continue to enroll directly
- Navigation: For paid courses, redirects to `/course-payment` with course details

```javascript
if (course.isPaid && course.price > 0) {
  router.push(`/course-payment?courseId=${id}&title=${encodeURIComponent(course.title)}&price=${course.price}&currency=${course.currency || 'INR'}`);
  return;
}
```

### 2. Created Payment Page
**File:** `jobzeeMobile/app/course-payment.jsx`

A dedicated payment page that handles the entire payment flow:

#### Features:
- **Secure Payment UI**: Clean, modern interface with course details and pricing
- **Razorpay Integration**: Opens native Razorpay checkout
- **Payment Verification**: Backend verification of payment signatures
- **Success Handling**: Automatic enrollment after successful payment
- **Error Handling**: User-friendly error messages for failures

#### Payment Flow:
1. **Create Order**: POST to `/api/learning/courses/create-payment-order`
2. **Open Razorpay**: Native Razorpay checkout with order details
3. **Payment Callback**: Razorpay returns payment details
4. **Verify Payment**: POST to `/api/learning/courses/verify-payment`
5. **Complete Enrollment**: User redirected to course details

#### UI Components:
- Course title and details
- Price display with currency
- Security badge (Razorpay powered)
- Feature list (access, materials, certificate)
- Secure payment button
- Loading states
- Disclaimer text

## Backend Endpoints Used

### Create Payment Order
```
POST /api/learning/courses/create-payment-order
Body: { courseId, amount, currency }
Returns: { order: { razorpayKeyId, razorpayOrderId, amount, currency, userEmail, userName, userPhone } }
```

### Verify Payment
```
POST /api/learning/courses/verify-payment
Body: { courseId, razorpay_order_id, razorpay_payment_id, razorpay_signature }
Returns: { success: true, enrollment: {...} }
```

## Payment Options Configuration

The Razorpay checkout opens with these pre-configured options:
- **Key**: Razorpay API key from backend
- **Amount**: Course price in smallest currency unit (paise for INR)
- **Order ID**: Generated by backend
- **Currency**: INR (default) or course-specific currency
- **Name**: "Jobzee Learning"
- **Description**: "Enrollment for [Course Title]"
- **Theme Color**: #6366F1 (indigo)
- **Prefill**: User email, phone, and name (from backend)

## Important Notes for Deployment

### Expo Development Build Required
Since `react-native-razorpay` uses native modules, you need to create a development build:

```bash
# Install EAS CLI if not already installed
npm install -g eas-cli

# Login to Expo
eas login

# Configure the project
eas build:configure

# Create a development build
eas build --profile development --platform android
# or for iOS
eas build --profile development --platform ios
```

### Alternative: Using Expo Go (Not Recommended for Production)
If you want to test without building, you can use WebView approach, but this is not recommended for production as it provides inferior UX.

### Razorpay Configuration
Ensure backend has:
1. Razorpay API Key ID
2. Razorpay API Key Secret
3. Webhook endpoints configured (for payment status updates)

## Error Handling

The payment page handles:
- **Payment Cancelled**: User friendly message when user cancels
- **Payment Failed**: Error description from Razorpay
- **Order Creation Failed**: Backend error messages
- **Verification Failed**: Prompts user to contact support
- **Network Errors**: Generic error messages

## Security Features

1. **Backend Verification**: All payments verified server-side using Razorpay signature
2. **Secure Order Creation**: Order created on backend with authentication
3. **No Client-Side Price Manipulation**: Amount and currency from backend
4. **SSL/TLS**: All API calls over HTTPS
5. **Token Authentication**: User must be logged in

## Course Payment vs Free Enrollment

| Feature | Free Course | Paid Course |
|---------|-------------|-------------|
| Enrollment | Direct | Via Razorpay |
| Backend Call | `/enroll` | `/create-payment-order` → `/verify-payment` → `/enroll` |
| User Flow | 1 step | 3 steps |
| Verification | Email only | Payment signature |
